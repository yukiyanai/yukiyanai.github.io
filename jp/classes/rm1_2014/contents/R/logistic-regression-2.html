数値計算：アルゴリズムによって最尤推定値を探す

二分法 (bisection method)

二分法で最尤推定値 (MLE) \(\hat{\theta}\) を求める実践的方法。

\[l(\theta)=\log L(\theta)\] とする。

  1. 対数尤度関数をグラフにし、\(\theta_L < \hat{\theta} < \theta_U\) となるような\(\theta_L\) と\(\theta_U\) を見つける（つまり、最大値がどの範囲にあるか目測で探す）。
  2. \(\theta_L\) と\(\theta_U\) の中間点 \(\theta_M = (\theta_L + \theta_U)/2\) を求める。
  3. \(l'(\theta_M) > 0\) なら\(\theta_L\) を \(\theta_M\) で置き換える。\(l'(\theta_M) > 0\) なら\(\theta_U\) を \(\theta_M\) で置き換える。（\(l'(\theta_M) = 0\)が見つかればその時点で終了。）
  4. 2に戻って操作を繰り返し、\(|l(\theta_m) - max(l(\theta_L), l(\theta_U))| < \epsilon\) になった時点で操作を止める。
  • 長所：連続な対数尤度関数に対し手順1を適切に行えば、必ず収束する
  • 短所：収束が遅い

例として、尤度関数 \[L(\theta) = \theta^{789} (1-\theta)^{211}\] を考える。 対数尤度は、 \[\log L(\theta) = 789 \log \theta + 211 \log (1-\theta)\] である。 この問題については、\(S(\theta)=0\) を解くことにより、\(\hat{\theta} = 0.789\) であることは簡単にわかるが、あえて二分法で最大値を求めてみよう。

まず、対数尤度関数を定義する。

l.lik.0 <- function(theta){
    l <- 789*log(theta) + 211*log(1 - theta)
    return(l)
}

次に、二分法を実行するための関数を用意する。

## 1階微分を計算する関数
df <- function(f, x){
    h <- 0.0001
    d <- (f(x + h) - f(x))/h
    return(d)
}
## 二分法
mle.bisection <- function(theta.l, theta.u, ll, criterion=0.1^10){
    ## 二分法で最尤推定値theta.hatを求める関数
    ## 引数：  theta.l = 最尤推定値の下限
    ##         theta.u = 最尤推定値の上限
    ##         ll = 対数尤度関数
    ##         criterion = 収束の基準（小さいほど正確）
    ## 返り値：theta.hat = 最尤推定値
    
    counter <- 1  ## 繰り返し回数を表示するためのcounter
    while(TRUE){  ## while ループで操作を繰り返す
        theta.m <- (theta.l + theta.u)/2
        l.lower <- ll(theta.l)
        l.upper <- ll(theta.u)
        l.med <- ll(theta.m)
        first <- df(ll, theta.m)
        
        if(first==0){
            l.max <- l.med
            break   
        }else if(first > 0){
            theta.l <- theta.m
            epsilon <- l.med - l.upper
        }else{
            theta.u <- theta.m
            epsilon <- l.med - l.lower
        }
        
        ## この時点までにみつけた対数尤度の最大値を表示する
        l.max <- max(c(l.lower, l.med, l.upper))
        cat("Iteration", counter, ": log likelihood =", l.max, "\n")
        
        ## 基準を満たした時点でループから抜ける
        if(abs(epsilon) < criterion) break
        counter <- counter + 1
    }
    
    ## 尤度を最大にするthetaの値theta.hat を見つけ、それを返す
    if(l.lower==l.max){
        theta.hat <- theta.l
    }else if(l.med==l.max){
        theta.hat <- theta.m
    }else{
        theta.hat <- theta.u
    }
    return(theta.hat)
}

ニュートン-ラフソン法 (Newton-Raphson Method)

ニュートン・ラフソン法で最尤推定値 (MLE) \(\hat{\theta}\) を求める実践的方法。

  1. 対数尤度関数をグラフにし、\(\hat{\theta}\)が大体どのあたりにあるか、予測する。
  2. 予測した値を初期値\(\theta\)とする
  3. \(\bar{\theta} \leftarrow \theta\) とおき、次のように\(\theta\)を更新する： \[\theta \leftarrow \bar{\theta} - \frac{l'(\bar{\theta})}{l''(\bar{\theta})}\]
  4. 3に戻って操作を繰り返し、\(|\theta - \bar{\theta}| < \delta\)になった時点で操作を止める。
  • 長所：収束が速い
  • 短所：必ず最大値を見つけるとは限らない

例として、先ほどと同じ関数の最尤推定値を求めよう。 まず、アルゴリズムを作る。

## 二階微分を計算する関数
## 一階微分の関数 df を利用
ddf <- function(f, x){
    h <- -0.0001
    return((df(f, x+h) - df(f, x))/h)
}
newton.mle <- function(ll, delta=0.1^12, theta.0) {
    h <- 0.001
    theta.bar <- theta.0
    counter <- 1
    while(TRUE) {
        first <- df(ll, theta.bar)
        second <- ddf(ll, theta.bar)
        theta.0 <- theta.bar - first/second
    
        ## この時点までにみつけた対数尤度の最大値を表示する
        l.max <- ll(theta.0)
        cat("Iteration", counter, ": log likelihood =", l.max, "\n")
        
        ## 基準を満たした時点でループから抜ける
        if (abs(theta.0 - theta.bar) < delta) break
            
        theta.bar <- theta.0
        counter = counter + 1
    }
    return(theta.0)
}

このアルゴリズムで、最尤推定値を求めてみよう。 初期値は0.5にしてみる。

newton.mle(l.lik.0, theta.0=0.5)
## Iteration 1 : log likelihood = -515.2786 
## Iteration 2 : log likelihood = -515.2786 
## Iteration 3 : log likelihood = -515.2786
## [1] 0.78895

先ほどと同様の結果が、先ほどより速く得られることがわかる。

母数が2つ以上の場合

母数が2変数以上の場合も同様のアルゴリズムで最大値を探すことができるが、プログラムが少々面倒なので、既存の関数を使おう。 最尤推定値を求める場合には、maxLik パッケージのmaxLik()を利用すればよい。 必要な引数は、対数尤度関数と、推定の初期値、使用する方法である。 対数尤度関数は、第1引数が母数ベクトルになるように設定する。 また、推定の初期値 (start) には、母数ベクトルと同じ長さの数値ベクトルを渡す。 ニュートン・ラフソン法を使うには、method="NR"とする。 実際に使ってみよう。

maxLik(l.lik.0, start=0.5, method="NR")
## Maximum Likelihood estimation
## Newton-Raphson maximisation, 2 iterations
## Return code 1: gradient close to zero
## Log-Likelihood: -515.2786 (1 free parameter(s))
## Estimate(s): 0.789

このように、\(\theta =0.5\)というあまり正確でない値から出発しても、素早く（繰り返しは2回！）最尤推定値を求めることができる。


ロジスティック回帰

例1：過去の当選回数で当選確率を予測する

浅野・矢内 (2013) の第14章に登場する架空のデータ（ml_wl.dta）を利用して説明する。

使用するデータを読み込む。

require(foreign)
myd <- read.dta("data/ml_wl.dta")

このデータを使い、自民党候補の小選挙区における当落 (wlsmd) を縦軸に、それまでの当選回数 (previous) を横軸にした散布図を表示してみよう。

p1 <- ggplot(myd, aes(previous, wlsmd)) + geom_point()
print(p1)

このデータを使い、当落を応答変数、過去の当選回数を説明変数として、ロジスティック回帰分析を行おう。 ここでは、glm() を使わず、自分で尤度関数を定義して最尤推定値を求めてみよう。 まず、対数尤度関数（講義スライドを参照）を定義する。 maxLik() で最尤推定値を求めるために、関数の第1引数は母数ベクトルにする。

terms <- 0:7
n.cand <- c(3, 2, 1, 2, 3, 3, 0, 1)
y <- c(1, 1, 0, 1, 2, 2, 0, 1)
ll.1 <- function(beta){
    N <- length(y)
    beta1 <- beta[1]
    beta2 <- beta[2]
    ll.vec <- rep(NA, N)
    for(i in 1:N){
        prob <- 1/(1 + exp(-beta1 - beta2*terms[i]))
        l <- prob^y[i] * (1 - prob)^(n.cand[i] - y[i])
        ll.vec[i] <- log(l)
    }
    ll <- sum(ll.vec)
    return(ll)
}

この対数尤度関数を使い、最尤推定値を求めよう。 推定する母数は2つあるので、初期値には長さ2の数値ベクトルを与える。

fit.01 <- maxLik(ll.1, start=c(0, 1), method="NR")
summary(fit.01)
## --------------------------------------------
## Maximum Likelihood estimation
## Newton-Raphson maximisation, 5 iterations
## Return code 1: gradient close to zero
## Log-Likelihood: -9.404832 
## 2  free parameters
## Estimates:
##      Estimate Std. error t value Pr(> t)
## [1,] -0.90945    0.96433 -0.9431  0.3456
## [2,]  0.36210    0.27850  1.3002  0.1935
## --------------------------------------------

これで最尤推定値、\(\hat{\beta}_1 \approx -0.91\)、\(\hat{\beta}_2 \approx 0.36\)を求めることができた。

AICを求めると、

-2 * summary(fit.01)$loglik + 2 * 2
## [1] 22.80966

となる。

次に、予測の正解率を求める。 そのために、まず予測値（予測確率）を求める。

例2：選挙費用で当選確率を予測する

自民党候補の小選挙区における当落 (wlsmd) を縦軸に、それまでの当選回数 (previous) を横軸にした散布図を表示してみよう。

p2 <- ggplot(myd, aes(expm, wlsmd)) + geom_point()
print(p2)

このデータを使い、当落を応答変数、過去の当選回数を説明変数として、ロジスティック回帰分析を行おう。 先ほどと同じように、glm() を使わず、自分で尤度関数を定義して最尤推定値を求めてみよう。 まず、対数尤度関数（講義スライドを参照）を定義する。 maxLik() で最尤推定値を求めるために、関数の第1引数は母数ベクトルにする。

r <- myd$wlsmd
x <- myd$expm
ll.2 <- function(beta){
    N <- length(r)
    beta1 <- beta[1]
    beta2 <- beta[2]
    ll.vec <- rep(NA, N)
    for(i in 1:N){
        prob <- 1/(1 + exp(-beta1 - beta2*x[i]))
        l <- prob^r[i] * (1 - prob)^(1 - r[i])
        ll.vec[i] <- log(l)
    }
    ll <- sum(ll.vec)
    return(ll)
}

この対数尤度関数を使い、最尤推定値を求めよう。 推定する母数は2つあるので、初期値には長さ2の数値ベクトルを与える。

fit.02 <- maxLik(ll.2, start=c(0, .5), method="NR")
summary(fit.02)
