問題１

Q1.階乗を計算する関数を作りなさい。


1. 関数の名前を“fan_n”とし、引数をnと定義する。

2. 戻り値は“result”とし、後にかけ算を行うため、“result”に“1”を割り当てる。

3. nが負の値を取らないと仮定しているため、n<0の場合、計算しないように設定を行う。

4. また、n=0の時、n!=1のため、この設定も行う。

fan_n<- function(n){
                      #引数はnとする。
                      #戻り値はresultと置き、それを１と定義する。
  result<- 1
  
  #n<０の場合
       if(n<0){       #nが０より上の値を取る場合、以下の表示をさせる。
          print(nが０より上の値を取る)
          }

  #n=０の場合
  else if(n==0){      #n=0の場合、結果は１になるので、戻り値を1に設定する。
                return(1)
                }
  #n>0の場合
  else{
    for(i in n:1){     #n>0の場合、resultにある数nから1を掛けたものをresultとして指定する。
      result<- result*i
                 }
    return(result)
       }
}

次に、確認のため、Rの内蔵関数(factorial)と自作の(fan_n)の結果を比較する。

factorial(5)==fan_n(5)
## [1] TRUE

結果についてTRUEであるので、自作の関数は内蔵関数と一致すると解釈できる。


問題2

Q2.自分で作った階乗の関数を利用して、組み合わせを計算する関数を作りなさい。


1. あるものをn個の中からk個を選ぶ組み合わせを計算するため、関数を“cho_n”と指定し、引数をn、kとする。

2. nとkの関係について場合分けを行う。

3. n>kを満たす組み合わせは存在しないので、戻り値を０とし、その他の場合は計算を行うように設定する。

cho_n<- function(n,k){ #引数はnとkであり、戻り値はresultである。
  if(n<k){             #n個の中からk個を選ぶ組み合わせは存在しないので、０と定義する。
  return(0)            #resultを0として返す。
  }
  
  else{                #n>kの場合、組み合わせの関数は以下の通りになる。
    result<- fan_n(n)/(fan_n(k)*fan_n(n-k))
    return(result)     #resultを返す
  }
}


次に、確認のため、Rの内蔵関数(choose)と自作のcho_nの結果を比較する。

choose(8,2)==cho_n(8,2)
## [1] TRUE


問題３

Q3.自分で作った組み合わせの関数を利用して、二項分布の確率質量を求める関数を作りなさい。


1. 二項分布の確率質量を求める自作の関数を“db_n”とし、引数をx,n,pとする。

2. 確率質量を求めるために計算を行う。その際、全問で自作した組み合わせ関数“cho_n”を利用する。

3. 上記の結果をresultに定義し、戻り値として設定する。

db_n<- function(x,n,p){ #引数はx,n,pであり、各々は成功回数、試行回数、成功確率を指す。
　　　　　　　　　　　　#確率質量の関数は以下の通り。
  result<- cho_n(n,x)*p^x*(1-p)^(n-x) 
  　　　　　　　　　　　
  return(result)        #戻り値はresultにし、それを返す。
}


次に、確認のため、Rの内蔵関数(dbinom)と作成した“db_n”の結果を比較する。

dbinom(2,100,0.7)==db_n(2,100,0.7) #x=2, n=100, p=0.7において、Rにある関数と上記の自作関数を比較。
## [1] FALSE


結果として、内蔵関数と自作した関数の値は一致しなかった。念のため、個別的に検証する。

dbinom(2,100,0.7) #既存の関数について、各々に先述の値を代入。
## [1] 1.388942e-48
db_n(2,100,0.7)   #自作の関数について、各々に先述の値を代入。
## [1] 1.388942e-48

小数点6位まで一致しており、それ以下の点で差が見られているだけであると推測できる。
